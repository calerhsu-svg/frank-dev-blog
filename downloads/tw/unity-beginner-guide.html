第一章：Unity 安裝與環境設定

Unity 是目前全球最主流的遊戲引擎之一，支援 2D/3D 遊戲開發、AR/VR、模擬應用等多種場景。本章將手把手帶你從零開始安裝 Unity Hub 與 Unity Editor。

1.1 下載 Unity Hub

前往 Unity 官網（unity.com）下載 Unity Hub。Hub 是管理 Unity 版本與專案的統一工具，建議使用最新的 LTS（長期支援版本），穩定性最高。

💡 建議：初學者請選 LTS 版本（如 2022.3 LTS），而非 Tech Stream，避免遇到不穩定的新功能。

1.2 建立第一個專案

安裝完成後，在 Unity Hub 中點選「New Project」，選擇 2D 或 3D 模板：

專案設定建議：
├── 專案名稱：MyFirstGame
├── 模板：2D (Built-in Render Pipeline)
├── 位置：桌面/UnityProjects/
└── Unity 版本：2022.3 LTS

目錄結構自動建立：
Assets/          ← 所有遊戲資源
Library/         ← Unity 快取（不要手動修改）
Packages/        ← 套件管理
ProjectSettings/ ← 專案設定

1.3 介面初探

打開專案後你會看到五個主要面板：

  • Scene：場景編輯視窗，拖拉物件、調整位置
  • Game：遊戲預覽視窗，按 Play 即可測試
  • Hierarchy：物件階層，顯示場景中所有 GameObject
  • Inspector：屬性面板，修改選中物件的 Component
  • Project：資源管理，管理所有檔案和素材
💡 面板可以自由拖拉重新排列。建議花 10 分鐘先熟悉預設版面配置。

第二章：介面導覽與基本操作

熟悉 Unity 編輯器的操作是提高開發效率的基礎。本章介紹最常用的快捷操作和工作流程。

2.1 場景導覽

場景視窗操控：
  滑鼠中鍵拖拉    → 平移視角
  滾輪             → 縮放
  Alt + 左鍵拖拉   → 旋轉視角（3D）
  F 鍵             → 聚焦選中物件
  雙擊 Hierarchy   → 聚焦該物件

2.2 Transform 工具

Scene 視窗左上角有四個變換工具：

  • W：移動工具（Move）— 拖拉箭頭移動物件
  • E：旋轉工具（Rotate）— 拖拉圓環旋轉物件
  • R：縮放工具（Scale）— 拖拉方塊縮放物件
  • T：矩形工具（Rect）— UI 元素專用

2.3 Play 模式

按下頂部工具列的 Play 按鈕（或 Ctrl+P）進入遊戲模式。在 Play 模式中的修改不會被保存！這是新手常踩的坑。

⚠️ 注意：Play 模式下的任何修改在停止後都會消失！養成習慣先停止再修改。

第三章：C# 程式設計基礎

Unity 使用 C# 作為主要程式語言。如果你有任何程式基礎（JavaScript、Python 等），C# 的學習曲線不會太陡。

3.1 第一個腳本

在 Project 面板右鍵 → Create → C# Script，命名為 PlayerController：

using UnityEngine;

public class PlayerController : MonoBehaviour
{
    // 公開變數會顯示在 Inspector 面板
    public float speed = 5f;
    public float jumpForce = 10f;

    // Start 在第一幀之前呼叫一次
    void Start()
    {
        Debug.Log("遊戲開始！");
    }

    // Update 每幀呼叫一次
    void Update()
    {
        // 取得水平輸入（左右鍵 / A D 鍵）
        float h = Input.GetAxis("Horizontal");

        // 移動角色
        transform.Translate(Vector3.right * h * speed * Time.deltaTime);

        // 按空白鍵跳躍
        if (Input.GetKeyDown(KeyCode.Space))
        {
            GetComponent<Rigidbody2D>().AddForce(Vector2.up * jumpForce, ForceMode2D.Impulse);
        }
    }
}

3.2 重要概念

  • MonoBehaviour：所有 Unity 腳本的基礎類別
  • Start()：初始化，只跑一次
  • Update()：每幀執行，放遊戲邏輯
  • Time.deltaTime：幀間隔時間，確保不同 FPS 下移動速度一致
  • public 變數：會出現在 Inspector，可在編輯器中調整

3.3 常用 API

// 取得 Component
Rigidbody2D rb = GetComponent<Rigidbody2D>();

// 查找物件
GameObject player = GameObject.Find("Player");
GameObject enemy = GameObject.FindWithTag("Enemy");

// 生成與銷毀
Instantiate(prefab, position, rotation);
Destroy(gameObject);          // 銷毀自己
Destroy(gameObject, 2f);      // 2 秒後銷毀

// 定時呼叫
InvokeRepeating("SpawnEnemy", 1f, 2f);  // 1秒後開始，每2秒執行
StartCoroutine(MyCoroutine());           // 協程

第四章：GameObject 與 Component

Unity 的核心哲學是「一切皆 GameObject，功能靠 Component」。理解這個概念是掌握 Unity 的關鍵。

4.1 GameObject

場景中的每個東西都是 GameObject：角色、敵人、子彈、攝影機、燈光、UI 按鈕，甚至空物件。GameObject 本身只是一個容器，真正的功能來自於附加在上面的 Component。

4.2 常用 Component

  • Transform：位置、旋轉、縮放（每個 GO 必有）
  • SpriteRenderer：2D 圖片渲染
  • Rigidbody2D：2D 物理（重力、力、速度）
  • Collider2D：2D 碰撞偵測（BoxCollider2D, CircleCollider2D）
  • AudioSource：播放音效
  • Animator：控制動畫狀態機

4.3 Prefab 預製物件

把設定好的 GameObject 從 Hierarchy 拖到 Project 面板，就會建立 Prefab。Prefab 像是「模板」——修改 Prefab 會自動更新所有實例。子彈、敵人、特效都應該做成 Prefab。

// 用程式生成 Prefab
public GameObject bulletPrefab;  // 在 Inspector 中指定

void Shoot()
{
    GameObject bullet = Instantiate(bulletPrefab, transform.position, Quaternion.identity);
    bullet.GetComponent<Rigidbody2D>().velocity = Vector2.right * 10f;
    Destroy(bullet, 3f);  // 3秒後自動銷毀，避免記憶體洩漏
}

第五章：物理系統入門

Unity 內建了強大的物理引擎。2D 遊戲使用 Physics2D 系統，只需要加上 Rigidbody2D 和 Collider2D 就能實現真實的物理效果。

5.1 設定物理

設定角色物理：
1. 選中 Player GameObject
2. Add Component → Rigidbody2D
   - Body Type: Dynamic（受力影響）
   - Gravity Scale: 1（正常重力）
   - Freeze Rotation Z: ✅（防止旋轉）
3. Add Component → BoxCollider2D
   - 調整 Size 符合角色大小

設定地面：
1. 選中 Ground GameObject
2. Add Component → BoxCollider2D（不需要 Rigidbody）

5.2 碰撞偵測

// 碰撞回呼（兩個物件都有 Collider，至少一個有 Rigidbody）
void OnCollisionEnter2D(Collision2D collision)
{
    if (collision.gameObject.CompareTag("Enemy"))
    {
        Debug.Log("碰到敵人！");
        // 處理受傷邏輯
    }
}

// 觸發器回呼（Collider 勾選 Is Trigger）
void OnTriggerEnter2D(Collider2D other)
{
    if (other.CompareTag("Coin"))
    {
        score += 10;
        Destroy(other.gameObject);
    }
}
💡 Collision = 物理碰撞（會彈開）；Trigger = 觸發器（穿過去但偵測到）。撿金幣、進入區域用 Trigger。

第六章：UI 系統

Unity 的 UI 系統基於 Canvas。所有 UI 元素（按鈕、文字、圖片）都必須是 Canvas 的子物件。

6.1 建立 UI

建立 UI 步驟：
1. Hierarchy 右鍵 → UI → Canvas
   - Canvas Scaler → Scale With Screen Size
   - Reference Resolution: 1920 x 1080
2. Canvas 右鍵 → UI → Text - TextMeshPro
   - 輸入 "Score: 0"
3. Canvas 右鍵 → UI → Button - TextMeshPro
   - 子物件 Text 改為 "開始遊戲"

6.2 用腳本控制 UI

using UnityEngine;
using TMPro;

public class UIManager : MonoBehaviour
{
    public TextMeshProUGUI scoreText;
    public GameObject gameOverPanel;

    private int score = 0;

    public void AddScore(int points)
    {
        score += points;
        scoreText.text = "Score: " + score;
    }

    public void ShowGameOver()
    {
        gameOverPanel.SetActive(true);
        Time.timeScale = 0f;  // 暫停遊戲
    }

    public void RestartGame()
    {
        Time.timeScale = 1f;
        SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex);
    }
}

第七章：動畫系統

Unity 的 Animator 系統使用狀態機控制動畫切換。2D 遊戲常用 Sprite Animation（圖序動畫）。

7.1 建立 Sprite 動畫

步驟：
1. 選中角色 → Window → Animation → Animation
2. Create → 命名 "PlayerIdle.anim"
3. 把圖片序列拖到時間軸
4. 調整 Samples（幀率），通常 8-12 FPS

常見動畫狀態：
Idle → Run → Jump → Fall → Idle（循環）

7.2 Animator Controller

// 透過程式控制動畫切換
Animator anim = GetComponent<Animator>();

// 設定參數
anim.SetFloat("Speed", Mathf.Abs(h));     // 移動速度
anim.SetBool("IsGrounded", isGrounded);    // 是否著地
anim.SetTrigger("Jump");                    // 觸發跳躍動畫
anim.SetTrigger("Attack");                  // 觸發攻擊動畫

第八章：音效與粒子特效

8.1 音效播放

public class SoundManager : MonoBehaviour
{
    public static SoundManager Instance;
    public AudioClip jumpSound;
    public AudioClip coinSound;
    public AudioClip bgm;

    private AudioSource audioSource;

    void Awake()
    {
        Instance = this;
        audioSource = GetComponent<AudioSource>();
        audioSource.clip = bgm;
        audioSource.loop = true;
        audioSource.Play();
    }

    public void PlaySFX(AudioClip clip)
    {
        audioSource.PlayOneShot(clip);  // 不中斷 BGM
    }
}

// 其他腳本中呼叫
SoundManager.Instance.PlaySFX(SoundManager.Instance.jumpSound);

8.2 粒子特效

Hierarchy 右鍵 → Effects → Particle System。常用場景：爆炸、煙霧、撿取金幣閃光、角色著地灰塵。

// 在碰撞時播放粒子特效
public GameObject explosionPrefab;

void OnDestroy()
{
    Instantiate(explosionPrefab, transform.position, Quaternion.identity);
}

第九章：場景管理與切換

using UnityEngine.SceneManagement;

// 載入場景（需先在 Build Settings 中加入場景）
SceneManager.LoadScene("GameScene");           // 用名稱
SceneManager.LoadScene(1);                      // 用索引

// 重新載入當前場景
SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex);

// 非同步載入（適合大場景 + Loading 畫面）
StartCoroutine(LoadAsync("GameScene"));

IEnumerator LoadAsync(string sceneName)
{
    AsyncOperation op = SceneManager.LoadSceneAsync(sceneName);
    while (!op.isDone)
    {
        float progress = Mathf.Clamp01(op.progress / 0.9f);
        loadingBar.value = progress;  // 更新進度條
        yield return null;
    }
}

第十章：Build 打包與發布

10.1 打包步驟

File → Build Settings
1. 加入所有場景（Scenes In Build）
2. 選擇平台：
   - PC/Mac → Build And Run
   - Android → 需安裝 Android SDK
   - iOS → 需要 Mac + Xcode
3. Player Settings 設定：
   - Company Name / Product Name
   - Icon
   - Resolution
   - Bundle Identifier (com.你的名字.遊戲名)
4. 點 Build

10.2 手機遊戲注意事項

  • Texture Compression：Android 用 ETC2，iOS 用 ASTC
  • Target API Level：Android 至少 API 33
  • 最低 iOS 版本：建議 iOS 14+
  • 測試多種解析度和機型
